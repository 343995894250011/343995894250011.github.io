Redis基础

基本操作

  • set key value：信息添加
  • get key：信息查询

数据类型

redis 自身是一个 Map 其中所有的数据都是采用key:value的形式储存

数据类型指的是存储的数据类型，也就是value部分，key部分永远是字符串

  1. string：单个数据，一个储存空间保存一个数据，如果字符串以整数的形式展示，可以作为数组操作使用

    基本操作

    • set key value
    • get key
    • del key
    • mset key1 value1 key2 value2 … 添加多个数据
    • mget key1 key2 … 获取多个数据
    • strlen key 获取数据的字符个数
    • append key value 如果存在原始信息则追加，否则新建

    扩展操作

    • 设置数值数据增加指定范围的值

      incr key

      incrby key increment

      incrby float key increment

    • 设置数值数据减少指定范围的值

      decr key

      decrby key increment

    • 设置数据具有指定的生命周期

      setex key seconds value

      psetex key milliseconds value

    从上面我们可以看到 string 在 redis 内部储存默认就是一个字符串，当遇到递增递减操作时会转换成数值类型进行计算，redis的所有操作都是原子性的，采用单线程处理所有业务，命令是一个一个执行的，因此无需考虑并发所带来的影响

    redis 可以用于控制数据库表主键 id ，为数据库表主键提供生成策略，保障数据库表主键的一致性。此方案适用于所有数据库，且支持数据库集群

    redis 控制数据的生命周期，通过数据是否失效控制业务行为

    redis 应用于各种结构型和非结构型高热度数据访问加速

  2. hash：对一系列数据进行编组，方便管理，典型应用：储存对象信息，底层适用哈希表

    如果 field 数量比较少，储存结构优化为数组结构，反之则使用 hash 结构

    基本操作

    • hset key field value
    • hget key field
    • hgetall key
    • hdel key field1 [field2]
    • hmset key field1 value1 field2 value2 …
    • hmget key field1 field2 …
    • hlen key 获取哈希表中的字段数量
    • hexists key field 获取哈希表中是否存在指定值字段

    扩展操作

    • 获取所有字段名或字段值

      hkeys key

      hvals key

    • 设置指定字段的数值数据或增加指定范围的值

      hincrby key field increment

      hincrbyfloat key field increment

    hash 类型下的 value 只能储存字符串，不允许储存其他类型数据，不存在嵌套现象，如果数据未获取到，对应值为 nil

    设计初衷不是为了存储大量对象而设计的，不能滥用，更不能将 hash 作为对象列表使用

    hgetall 操作可以获取全部属性，但是遍历整体数据效率较低，可能会成为数据访问瓶颈

  3. list：存储多个数据，并对数据进入存储空间的顺序进行区分，底层使用双向链表

    基本操作

    • lpush key value1 [value2]

    • rpush key value1[value2]

    • lrange key start stop

    • lindex key index

    • llen key

    • lpop key

    • rpop key

    扩展操作

    • 规定时间内获取并移除数据

      blpop key1 [key2] timeout

      brpop key1 [key2] timeout

      brpoplpush source destination timeout

    • 移除指定数据 lrem key count value

    应用场景：数据具有顺序特征，需要获取最新消息，或者解决多路信息汇总合并

  4. set：储存大量不重复数据，并且提供高查询效率

    基本操作

    • sadd key member1 [member2]
    • smembers key 获取全部数据
    • srem key member [member2] 删除数据
    • scard key 获取集合数据总量
    • sismember key member 判断集合中是否包含指定数据

    扩展操作

    • 随机获取集合中指定数量的数据

      srandmember key [count]

    • 随机获取集合中的某个数据并将该数据移出该集合

      spop key [count]

    • 求两个集合的交，并，差集

      sinter key1 [key2]

      sunion key1 [key2]

      sdiff key1 [key2]

    • 求两个集合的交，并，差集并储存到指定集合中

      sinterstore destination key1 [key2]

      sunionstore destination key1 [key2]

      sdiffstroe destination key1 [key2]

    • 将指定数据从原始集合中移动到目标集合中

      smove source destination member

  5. sorted_set：在 set 储存结构的基础上添加了可排序字段

    基本操作

    • zadd key score1 member1 [score2 member2]
    • zrange key start stop [withscores]
    • zrevrange key start stop [withscores]
    • zrem key member [member]
    • zrangebyscore key min max [withscores] [limit]
    • zrevrangebyscore key max min [withscores]
    • zremrangebyrank key start stop
    • zremrangebyscore key min max
    • zcard key
    • zcount key min max
    • zinterstore destination numkeys key [key]
    • zunionstore destination numkeys key [key]

    扩展操作

    • 获取数据对应的索引（排名）

      zrank key member

      zrevrank key member

    • score值获取与修改

      zscore key member

      zincrby key increment member
