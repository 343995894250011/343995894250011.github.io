PageDomain

封装分页相关参数，方便在业务中进行传递和处理

TableDataInfo

封装返回给前端的表格数据，包括总记录数、列表数据、消息状态码和消息内容。

TableSupport

  • TableSupport类是一个表格数据处理的工具类，主要用于封装分页参数。
  • 定义了常量用于表示分页参数的参数名，包括当前记录起始索引、每页显示记录数、排序列、排序方向、分页参数合理化等。
  • 提供了getPageDomain方法，该方法从请求中获取分页相关的信息，并创建一个PageDomain对象封装这些信息，然后返回。
  • buildPageRequest方法直接调用getPageDomain方法，用于构建分页请求。

BaseController

AjaxResult

统一请求成功和请求失败的返回信息

utils

ServletUtils

  • ServletUtils类是一个客户端工具类，提供了一系列用于处理HttpServletRequest、HttpServletResponse以及HttpSession的方法。
  • 包含了获取请求参数、获取请求对象、获取响应对象、获取Session对象等常用方法。
  • 提供了将字符串渲染到客户端、判断请求是否为Ajax异步请求、进行URL编码和解码等辅助方法。
  • 通过RequestContextHolder获取ServletRequestAttributes对象，从而获取当前的请求和响应对象。

PageUtile

SecurityUtils

这段代码是安全服务工具类，提供了一系列方法用于获取当前登录用户的信息，进行密码加密与匹配，以及权限验证等功能。主要功能包括：

  • 获取当前登录用户的ID、部门ID和用户名。
  • 获取当前登录用户的详细信息。
  • 生成密码的 BCrypt 加密字符串以及判断密码是否匹配。
  • 判断用户是否为管理员。
  • 验证用户是否具备某权限。
  • 验证用户是否拥有某个角色。

用户信息通常储存在认证信息（或者令牌）中，

HttpStatus

exception

这个包中的实现是，按几个大类根据相应场景有不同的自定义异常，有点繁琐。有另外一种思路是只定义一个自定义异常类，抛出时传入不同的状态码，通过不同的状态码获取到messages.properties中的不同错误信息，只要对状态码有一个统一定义规范就能有效区分不同异常进行管理

service

PermissionService

一个自定义的权限服务类，主要功能是通过 SecurityUtils 获取当前登录用户的信息，然后根据传入的权限字符串或角色字符串进行权限和角色的验证。通过设置权限上下文，将权限字符串传递给后续的权限验证，以确保权限验证的正确进行。此服务类是在 Spring Security 基础上进行的自定义扩展，提供了简便的方法用于权限和角色的判断。

  • 创建一个空的 Set<String> 类型的集合 perms，用于存储用户的权限信息。
  • 检查用户是否为管理员（isAdmin() 方法），如果是管理员，则直接将 "*:*:*"（表示所有权限）添加到 perms 集合中。
  • 如果用户不是管理员，获取用户的角色信息，然后根据角色获取权限：
    • 如果用户拥有多个角色，通过遍历用户的角色列表，为每个角色获取权限。
      • 调用 menuService.selectMenuPermsByRoleId(role.getRoleId()) 方法，从 menuService 中获取该角色对应的权限集合，并将这些权限设置到角色对象的 permissions 属性中。
      • 将角色的权限集合添加到总的权限集合 perms 中。
    • 如果用户没有角色，直接根据用户 ID 从 menuService 中获取用户的权限集合。
  • 返回最终的权限集合 perms。

exception

GlobalExceptionHandler

此全局异常处理器通过捕获不同类型的异常，提供了统一的异常处理策略，以及返回给前端的规范化错误信息

manager

factory

AsyncFactory

用于产生异步任务的异步工厂，该工厂类定义了两个方法，分别用于记录登录信息和操作日志的异步任务

AsyncManager

异步任务管理器

  1. 异步任务调度管理： 该类通过ScheduledExecutorService实现异步任务的调度，允许在指定时间或周期性地执行任务。
  2. 单例模式： 使用单例模式确保在整个应用程序生命周期内只有一个AsyncManager的实例，以便全局共享。
  3. 执行任务： 提供execute方法用于执行TimerTask类型的任务，这表示在指定时间执行的任务。在这里，任务将在延迟10毫秒后执行。
  4. 停止任务线程池： 提供shutdown方法用于停止任务线程池。这可以在应用程序关闭时调用，确保所有的异步任务都已完成，并且线程池已经关闭。

ShutdownManager

关闭管理器，确保应用退出的时候能关闭后台线程

system

SysLoginController
  1. 处理用户登录请求，验证用户身份，生成令牌返回给客户端。
  2. 获取当前登录用户的信息、角色集合和权限集合，用于前端展示和权限控制。
  3. 获取当前登录用户的菜单信息并构建成路由信息，用于前端动态路由的展示和权限控制。

common

CaptchaController

定义了一个Spring MVC REST控制器，负责根据配置（基于数学或字符）生成验证码图像，并在Redis中存储相应的验证码代码以供验证。控制器返回一个响应，包含UUID、Base64编码的验证码图像以及指示系统配置中是否启用验证码的标志。

CommonController

定义了一个Spring MVC REST控制器，提供了通用的文件上传、下载和本地资源下载功能。其中，文件上传支持单个文件和多个文件上传，文件下载支持普通文件下载和本地资源下载。

SysProfileController

个人信息控制类，可以获取个人信息和修改给人信息

monitor

集中存储与监控和度量相关的代码和配置，使得这些功能模块更易于维护和管理。
