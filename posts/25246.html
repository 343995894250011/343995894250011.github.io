GIT

git 工作流程

  • 本地仓库——开发人员自己电脑上的Git仓库（.git）文件夹
  • 远程仓库——远程服务器上的Git仓库
  • 工作区——写代码的地方
  • 暂存区——本地的特殊文件（index），临时储存我们即将要提交的文件

IDEA上继承的git使用

在某个分支上点击鼠标右键

  1. Edit Commit Message (编辑提交消息)：

    • 修改提交信息，用于纠正拼写错误，改进描述，调整提交信息

  2. Fixup… (修复提交)：

    • 标记为修复提交，修复提交会合并到其他前面的提交中

  3. Squash Into… (合并到…)：

    • 将选定的提交与前面的提交合并

  4. Drop Commit (丢弃提交)：

    • 丢弃提交，删除选定的提交，会对文件造成影响

  5. Interactively Rebase from Here… (从这里交互式地变基)：

    • 不太懂

  6. Push All up to Here… (推送到这里)：

    • 将所有提交推送到选定提交，用于将本地提交推送到远程仓库并覆盖远程仓库的提交历史

  7. Reset Current Branch to Here (重置当前分支到这里)：

  8. Revert Commit (还原提交)：

  9. Undo Commit (撤销提交)：

  10. Checkout (切换)：

    • 切换分支，可以在工作区中查看或修改特定的版本

  11. Revision / Show Repository at Revision (版本 / 显示指定版本的仓库)：

    • 查看仓库在特定版本时的状态

  12. Compare with Local (与本地比较)：

    比较之前仓库和本地仓库的版本之间的差异

一般流程

使用Git 与他人合作的一般工作流程可以概括为以下几个步骤：

  1. 克隆仓库 (Clone Repository):

    • 通过 git clone 命令从远程仓库中克隆代码到本地。

  2. 创建分支 (Create a Branch):

    • 在本地仓库中创建一个新的分支，用于开发新功能或修复 bug。

  3. 进行开发 (Development):

    • 在新创建的分支上进行代码的修改和开发。

  4. 提交更改 (Commit Changes):

    • 使用 git add 和 git commit 将修改的代码提交到本地仓库。

  5. 拉取更新 (Fetch and Pull):

    • 在提交更改之前，先拉取远程仓库的最新变更，确保本地代码与远程同步。

  6. 解决冲突 (Resolve Conflicts):

    • 如果在拉取更新时发生冲突，需要手动解决冲突。

  7. 推送更改 (Push Changes):

    • 将本地分支的更改推送到远程仓库。

  8. 创建合并请求 (Create a Pull Request):

    • 在远程仓库中创建一个合并请求（Pull Request，或简称 PR），请求将本地分支的更改合并到主分支或其他目标分支。

  9. 审查与合并 (Review and Merge):

    • 团队成员对合并请求进行审查，确认没有问题后，将其合并到目标分支。

  10. 删除本地分支 (Delete Local Branch):

    • 在合并后，可以删除本地分支。

  11. 删除远程分支 (Delete Remote Branch):

    • 如果需要，也可以删除远程分支。
