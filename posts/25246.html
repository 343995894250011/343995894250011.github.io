Git

起步

  • 什么是版本控制

    版本控制是一种记录一个或若干个文件内容变化，以便将来查阅特定版本修订情况的系统

  • 本地版本控制系统

    rcs：原理，在硬盘上保存补丁集（文件修订前后的变化），通过应用所有的补丁，可以重新计算出各个版本的文件内容

  • 集中化的版本控制系统

    为了让不同系统上的开发者协同工作而被产生。服务器宕机就凉凉

  • 分布式版本控制系统

    在这种系统中，客户端并不只是提取最新版本的文件快照，而是把代码仓库完整的镜像下来，这样一来，任何一处协同工作的服务器发生故障后，事后都可以用任何一个镜像出来的本地仓库恢复。因为每一次克隆操作，实际上都是一次对代码仓库的完整备份

Git

  1. 直接记录快照，而非差异比较

    Git 把数据看作是对小型文件系统的一组快照，每次提交更新，或者在Git中保存项目状态时，它主要对当前的全部文件制作一个快照并保存这个快照的索引。为了高效，如果文件没有被修改，Git不再重新储存该文件，而是只保留一个链接指向之前储存的文件。Git对待数据更像是一个快照流

  2. 几乎所有的操作都是本地执行

  3. Git保证完整性

    Git使用SHA-1散列。Git数据库中保存的信息都是以文件内容的哈希值来索引，而不是文件名

  4. Git 一般只添加数据

  5. 三种状态

    • 已提交（committed）

      表示数据已经安全的保存在本地的数据库中

    • 已修改（modified）

      表示已修改了文件，但还没保存到数据库中

    • 已暂存（staged）

      表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中

  6. 三个工作区域

    • Git 仓库

      Git仓库目录是Git用来保存项目元数据和对象数据库的地方

    • 工作目录

      是项目的某个版本独立提取出来的内容

    • 暂存区域

      一个文件，保存了下次将要提交的文件列表信息，一般在仓库目录中

  7. 工作流程

    • 在工作目录中修改文件
    • 暂存文件，将文件的快照放入暂存区域
    • 提交更新，找到暂存区域的文件，将快照永久性储存到Git仓库目录

仓库中文件的状态

  • 已跟踪

    被纳入版本更新的文件，

  • 未跟踪

    工作目录中除了已跟踪文件以外的所有其他文件都属于未跟踪文件，他们不存在于上次快照记录中，也没有存放入暂存区，初次克隆某个仓库的时候，工作目录中的所有文件都护属于已跟踪文件，并处于未修改状态

查看当前文件状态使用git status命令

跟踪新文件

暂存已修改的文件

在文件修改之后提交会提交最后一次运行 git add命令的那个版本，运行git add之后修订的文件，需要重新运行git add把最新版本重新暂存起来

使用 git status -s 可以得到另一种结构的输出

新添加到暂存区中的文件前面有A标记

修改过的文件前面有M标记——左边的M文件表示该文件被修改了并放入暂存区，右边的M表示该文件被修改了并放入了暂存区

？？表示新添加的未跟踪文件

忽略文件

如果不需要某些文件出现在跟踪文件列表中，可以创建一个名为.gitignore的文件

查看已暂存和未暂存的修改

git diff比较工作目录和当前文件暂存区域快照之间的差异，也就是修改之后还没有暂存起来的变化内容，只显示尚未暂存的改动，而不是自上次提交以来所作的所有改动

git diff –cached 查看暂存区与最近一次提交之间的差异，会显示已经添加到暂存区但尚未提交的文件的更改内容

git diff 查看工作目录中的文件与最近一次提交之间的差异，而git diff –cached用于比较暂存区中的文件与最近一次提交之间的差异

提交更新

首先确认是否全部提交到暂存区，然后执行 git commit，会将暂存区中的更改提交到版本库中git commit -m "commit message" - m用于指定提交消息，执行git commit 后，git 会将暂存区中的更改打包成一个新的提交，并将其保存到版本库中。每个提交都有一个唯一的哈希值，用于标识该提交，提交后的更改将成为版本库的一部分

  • 跳过使用暂存区

    git commit -a会自动把所有已经跟踪过的文件夹暂存起来一并提交

  • 移除文件

    从Git中移除某个文件，必须从已跟踪文件清单中移除，然后提交，使用git rm命令，并连带从工作目录中删除指定文件即可

    如果只是简单删除，那么在运行git status中就会在未暂存清单中看到

查看提交历史

git log如果不用任何参数，会按提交时间列出所有的更新，会列出每个提交的SHA-1校验，作者的名字和电子邮件地址，提交时间以及提交说明

-p显示每次提交内容的差异

-2显示最近两次提交

--stat提交简略的统计信息

撤销操作

  • git commit –amend 重新提交，可以覆盖原来的提交信息，最后只会有一次提交
  • git reset HEAD file 取消暂存
  • git checkout file 撤销之前做的修改

远程仓库的使用

  • git remote 列出一个远程服务器的简写

    -v 显示需要读写远程仓库使用的Git保存的简写对应的URL

  • git remote add shortname url 添加一个新的远程仓库，同时指定一个简写，可以使用该简写来代替整个URL，克隆默认有一个origin

  • git fetch name 拉取 name 仓库中有但你没有的信息

  • git push remote-name branch-name 推送 branch 到remote

  • git remote show reomte-name 查看一个远程仓库的信息

  • git remote rename 修改一个远程仓库的简写名

查看分支 git branch –contains

打标签

  • git tag <标签名><提交ID>列出已有的标签
  • git tag -a<标签名称> -m”<标签信息>”<提交ID>
  • git push origin<标签名称> 将标签推送到远程仓库
  • git checkout -b branchname 校准工作目录与从库中特定版本的标签名
